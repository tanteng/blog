MySQL 数据库设计及性能优化的一些规则和技巧。

<!--more-->

### 存储引擎选择

- 一般情况可以选择 MyISAM 存储引擎
- 如果需要事务支持必须使用 InnoDB 存储引擎
- MyISAM 索引有1000字节限制，InnoDB 数据按聚簇索引有序排列

### 命名规则

1. 数据库和表名应尽可能和所服务的业务模块名一致
2. 服务于同一个子模块的一类表应尽量以子模块名作为前缀或后缀
3. 字段名称也应尽量保持和实际数据相对应
4. 联合索引名称应尽量包含所有索引键字段名

### 字段类型选择

1. 经常需要计算和排序等消耗 CPU 的字段，应该尽量选择更为迅速的字段，如用 TIMESTAMP 代替 Datetime
2. 变长字段使用 varchar，不要使用 char
3. 对于二进制多媒体数据、流水队列数据不要放在数据库字段中

### 数据库索引

1. 业务需要的相关索引是根据实际的设计所构造 SQL 语句的 where 条件来确定的
2. 唯一确定一条记录的一个字段或多个字段要建立主键或者唯一索引
3. 业务通过不唯一索引访问数据时，需要考虑通过该索引值返回的记录稠密度
4. 需要联合索引的数据库要注意索引的顺序

### Query 语句优化

1. Insert 语句批量一次插入 1000 条时效率最高
2. 查询条件中各种逻辑操作符性能顺序是 and、or、in
3. 永远用小结果集驱动大记录集
4. 尽量使用 select * 只取需要的 columns
