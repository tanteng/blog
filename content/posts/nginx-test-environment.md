假如我们使用 git 进行版本控制，在一个大型网站中，开发人员在不同的分支上开发不同的需求，当一个需求开发完成需要测试，我们会把分支代码提供给测试人员。

设想一个场景：只有一个分支提测的时候，在测试机上可以直接拉取这个分支的代码进行测试，但是如果同时有多个分支都要同时进行测试，那么就没办法在一台测试机上同时进行了。

<!--more-->

### 实现原理

通过 Nginx 判断 User-Agent，不同的 UA 转发到不同的端口，每个端口下对应一个网站目录。

### Nginx 判断 UA

通过 Nginx 可以获取每次请求的 UA，即 $http_user_agent 变量。

```nginx
if ( $http_user_agent ~ "dashen" ) {
    set $flag "01";
}
```

### Nginx 反向代理和负载均衡

通过判断不同的 UA，我们可以通过反向代理转发到不同的机器和端口。
